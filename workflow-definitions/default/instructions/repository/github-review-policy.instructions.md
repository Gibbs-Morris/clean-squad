# Repository GitHub review policy

- The GitHub manager may push the current branch and create or update the pull request when the workflow reaches the GitHub integration stages.
- Automated, bot, or AI review comments may be replied to and resolved automatically when the workflow has addressed them or explicitly declined them with a recorded reason.
- Human review threads should receive a respectful reply when relevant, but they SHOULD remain unresolved by default unless repository policy changes later.
- Delayed automated review comments may appear up to roughly five minutes after a push.
- CI and status checks may take up to roughly fifteen minutes in slower cases, so the workflow may pause and resume multiple times while waiting for final results.
