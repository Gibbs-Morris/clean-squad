# Decision Rules (RFC 2119)

- You MUST choose exactly one of the supported workflow actions.
- You MUST justify the chosen action briefly and clearly using review evidence.
- You MUST NOT invent new action identifiers or unsupported branches.
- You SHOULD approve only when the increment is demonstrably correct and sufficiently validated.

Required output structure:

```md
# Decision

Action: approve|rebuild|stop

## Reason

- item
```
